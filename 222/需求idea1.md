#一. 漏洞库（Vulnerability Intelligence Center）

目标：实现漏洞全生命周期管理，支撑快速研判与修复。

    **自动收录新漏洞：**
        接入主流漏洞源（NVD、CNVD、CVE、GitHub Security Advisories、Exploit-DB、厂商公告等）。
        支持 RSS/API/Webhook 实时同步，每日增量更新。
    **重大漏洞预警推送：**
        对 0day/1day 或 CVSS ≥ 9.0 的漏洞触发实时告警。
        多通道通知（企业微信、钉钉、邮件、短信），支持按客户/资产/业务线定制推送策略。
    **漏洞分类分级：**
        自动打标：类型（RCE、XSS、SQLi、信息泄露等）、影响组件（如 Spring、Log4j）、适用场景（Web/云原生/工控）。
        动态风险评分 = CVSS × 资产暴露度 × 是否存在在野利用（EPSS）× 业务关键性。
    **漏洞利用方式（POC/EXP）：**
        提供验证性 POC（非攻击性），支持沙箱环境一键运行。
        标注检测规则（YARA/Sigma）、流量特征、WAF绕过技巧。
    **漏洞原理与修复方法：**
        AI 自动生成通俗易懂的漏洞原理说明。
        提供多维度修复建议：官方补丁链接、配置加固命令、临时缓解措施、WAF虚拟补丁规则。

#二. 工具箱（Security Toolbox）

目标：标准化、安全化、高效化使用安全工具。

    **按用途分类：**
        信息收集（如 sublist3r、theHarvester）
        漏洞扫描（如 Nuclei、Xray、Goby）
        渗透测试（如 Burp Suite、sqlmap）
        应急响应（如 Volatility、Sysmon 分析脚本）
        密码审计（如 John the Ripper、Hashcat）
        日志与流量分析（如 Zeek、Elasticsearch 查询模板）
    **工具收录与管理：**
        只收录经过安全团队验证的“好用、稳定、合规”工具。
        区分开源工具、商业工具、自研工具。
    **工具联动：**
        接入项目使用的开源/商业工具，接口直接调用使用。
    **提供使用方式：**
        每个工具附带：适用场景、典型命令、参数说明、输出解读、合规提示。
        支持 Web 化封装或 Docker 容器化，降低使用门槛。
    **操作审计与权限控制：**
        记录谁、何时、对哪些目标使用了什么工具。
        高危工具（如 mimikatz）需二次授权方可使用，满足等保审计要求。

#三. 资料库（Security Delivery Knowledge Hub）

目标：打通售前到交付全链路，实现知识复用与质量保障。

    **文档全周期覆盖：**
        售前阶段：解决方案PPT、技术白皮书、竞品对比表、报价模板、POC方案。
        实施阶段：项目计划书、测试方案、检查清单（Checklist）、验收标准。
        交付阶段：漏洞报告模板、渗透测试报告、基线核查表、整改建议书。
        运维阶段：SOP（标准操作流程）、应急预案、巡检报告、复盘总结。
        合规阶段：等保2.0/3.0材料包、GDPR/CCPA指南、行业监管要求汇编。
    **模板提供与内容优化：**
        所有文档提供标准化模板，支持一键生成初稿。
        AI辅助优化语言表达、逻辑结构、术语一致性。
        自动检测内容过时（如引用已废弃的漏洞编号或工具版本）。
    **协作与版本管理：**
        支持多人协同编辑、评论、审批流程。
        文档版本自动归档，支持差异对比。

#四. 需求编排（Service Orchestration Engine）

目标：将服务需求转化为可执行的标准化工作流。

    **输入服务类型，输出完整执行方案：**
        示例：输入“入网安全评估服务”，系统自动输出：
            前置条件：获取客户书面授权、签署保密协议。
            实施内容：外部渗透测试 + 内网漏洞扫描 + 主机/数据库基线核查。
            所需工具：Goby（资产发现）、Xray（漏洞扫描）、Burp Suite（Web渗透）、OpenSCAP（基线检查）。
            交付材料：《渗透测试报告》《漏洞汇总表》《安全加固建议书》《等保合规差距分析》。
            关联流程：立项 → 授权确认 → 扫描窗口预约 → 实施 → 报告审核 → 客户交付。
    **支持多种服务类型：**
        入网安评、等级保护测评、渗透测试、应急响应、代码审计、红蓝对抗、安全培训等。
    **动态适配客户环境：**
        根据客户行业（金融/政务/医疗）、资产规模、合规要求，自动调整流程细节。
    **与其它模块深度联动：**
        自动调用漏洞库获取最新风险项；
        推荐工具箱中的适用工具；
        引用资料库中的标准模板生成交付物。
